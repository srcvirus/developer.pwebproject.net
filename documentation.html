A pWeb home agent exposes a number of RESTful API for developing applications. The home agent REST API has the following format: http://<home_agent_fqdn>:<home_agent_port>/?method=method_name&parameters. The available methods are summerized in the following table. Then we describe each method in detail with their corresponding parameters and return values.

Method name Description
register_user Registers a new user with the home agent.
is_username_available Checks if a username is already registered with the home agent or not.
authenticate_user Authenticate a user against the provided username and password.
register_device Register a new device for a user.
update_device Update an already registered device's information.
is_devicename_available Checks if a device name is already registered against a given username.
delete_device Deletes a user's device identified by the devicename.
get_user_device Obtain a list of all devices belonging to a user.

I. User Device and Home Agent API

I(a). Register a new user (method=register_user)

Description

Registers a new user with the home agent.

Parameters

Parameter Name Required Description
username yes Expected user name for the new user to be create
password yes Password of the user
email yes A valid email address that will be used to verify the user
fullname no Actual name of the user
location no Home location for the user. usually city and country
affiliation no Affiliated organization for the new user

Example:

http://uw01.pwebproject.net:8088/?method=register_user&username=bob123&password=mypass&email=bob@uwaterloo.ca&fullname=Bob+Dylan&location=Waterloo,Ontario,Canada&affiliation=Univerity+of+Waterloo

Return Value

On Success: {"status":"success"}
On Failure: {"status":"success", "error":"<error_code>"}

Error Codes

Error Code Description
APP:7501 Username already exists
CDB:<number> Cassandra database related error

I(b) Check availability of a username (method=is_username_available)

Description

Checks if a username is already registered with the home agent or not.

Parameters

Parameter Name Required Description
username yes Username to check

Example:

http://uw01.pwebproject.net:8088/?method=is_username_available&username=bob123

Return Value

If username is available: {"status":"success", "availability":"yes"}
If username is not available: {"status":"success", "availability":"no", "suggestion" : ["<username1>, <username2>, ..."]}
On error: {"status":"error", "error":"<error_code>"}

Error Codes

Error Code Description
CDB:<number> Cassandra database related error

I(c). Authenticate a user (method=authenticate_user)

Description

Authenticate a user against the provided username and password.

Parameters

Parameter Name Required Description
username yes Username
password yes Password

Example:

http://uw01.pwebproject.net:8088/?method=authenticate_user&username=bob123&password=bobthebuilder

Return Value

If username and password is correct: {"status":"success", "authenticated":"yes"}
If username or password is not correct: {"status":"success", "authenticated":"no"}
On error: {"status":"error", "error":"<error_code>"}

Error Codes

Error Code Description
CDB:<number> Cassandra database related error

I(d) Register new device (method=register_device)

Description

Register a new device for a user.

Parameters

Parameter Name Required Description
devicename yes Name of the device
username yes username of the user
type no Type of the device (e.g., laptop)
ip yes IP address of the device
port yes Port number of the service running on the device
os no Operating System of the device
description no A description of the device
is_indexed yes Boolean value indicating whether the device will be publicly indexed (true) or not (false)

Example:

http://uw01.pwebproject.net:8088/?method=register_device&device_name=hammer&username=bob123&type=laptop&ip=192.168.1.25&port=8088&os=windows&is_indexed=true

Return Value

On Success: {"status":"success"}
On Error: {"status":"success", "error":"<error_code>"}

Error Codes

Error Code Description
APP:6401 Device name already exists for this user
APP:7502 User does not exist
CDB:<number> Cassandra database related error

Description

Parameters

Parameter Name Required Description

port no Port number of the service running on the device
os no Operating System of the device

On Success: {"status":"success"}
On Error: {"status":"success", "error":"<error_code>"}

Error Codes

Error Code Description
APP:6402 Device name does not exist
APP:7502 User does not exist
CDB:<number> Cassandra database related error

Checks if a device name is already registered against a given username.
